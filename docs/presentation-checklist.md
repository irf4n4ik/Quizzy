# Чек-лист презентации

## Слайд 1. Титульный
- Название проекта: `Quizzy`
- Подзаголовок: веб-приложение для проведения викторин в реальном времени
- ФИО автора или команды
- Дата и учебная группа, если требуется

## Слайд 2. Проблема и цель
- Какую задачу решает проект
- Почему квизы в реальном времени удобны для обучения, мероприятий и команд
- Цель: создать MVP, который позволяет запускать и проходить викторины онлайн

## Слайд 3. Соответствие ТЗ
- Регистрация и авторизация
- Создание и настройка квизов
- Вопросы разных типов
- Подключение по коду комнаты
- Игра в реальном времени
- Подсчёт баллов и лидерборд
- История игрока и организатора

## Слайд 4. Выбранный стек
- Клиент: `React + Vite + TypeScript`
- Сервер: `Node.js + Express`
- База данных: `PostgreSQL + Prisma`
- Реальное время: `Socket.IO`
- Контейнеризация: `Docker Compose`

## Слайд 5. Архитектура
- Клиент и сервер разделены
- Сервер хранит данные и управляет игрой
- PostgreSQL используется как основное хранилище
- Socket.IO синхронизирует состояние комнаты между участниками

## Слайд 6. Модель данных
- Пользователь
- Викторина
- Вопрос
- Вариант ответа
- Игровая комната
- Участник комнаты
- Ответ участника

Можно взять схему из `server/prisma/schema.prisma` и перенести её в диаграмму.

## Слайд 7. Демонстрация интерфейса
- Главная страница
- Экран регистрации и входа
- Кабинет организатора
- Редактор викторины
- Комната игрока
- Итоговый экран результатов

## Слайд 8. Демонстрация сценария
Показать по шагам:
1. Организатор входит в систему
2. Создаёт квиз
3. Запускает комнату
4. Игрок подключается по коду
5. Игра проходит в реальном времени
6. Лидерборд обновляется
7. История сохраняется

## Слайд 9. Что реализовано технически
- JWT-авторизация
- Проверка ролей
- CRUD для квизов
- Загрузка изображений к вопросам
- Серверный таймер вопросов
- Серверный подсчёт баллов
- История и статистика в личных кабинетах

## Слайд 10. Docker и запуск
- Проект запускается одной командой:

```bash
docker compose up --build
```

- Поднимаются `postgres`, `server`, `client`
- На первом запуске автоматически применяются миграции и сиды

## Слайд 11. Результат
- Получен работоспособный MVP
- Все основные функции из ТЗ реализованы
- Проект можно показать, протестировать и передать другому человеку для запуска

## Слайд 12. Перспективы развития

- Расширенная аналитика по играм
