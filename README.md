# Quizzy

Quizzy — это MVP-платформа для проведения квизов в реальном времени. Организатор создаёт викторины, запускает комнату по коду и управляет ходом игры, а игроки подключаются, отвечают на вопросы по таймеру и видят итоговый лидерборд.

## Ключевые возможности MVP
- регистрация и авторизация игроков и организаторов
- создание и редактирование викторин
- вопросы с одиночным и множественным выбором
- загрузка изображений к вопросам
- запуск комнаты и подключение по короткому коду
- проведение викторины в реальном времени через Socket.IO
- лидерборд и финальные результаты
- история игр в кабинетах игрока и организатора

## Сценарий демонстрации
1. Организатор входит в систему.
2. Открывает кабинет и создаёт или редактирует квиз.
3. Запускает комнату и получает код подключения.
4. Игрок входит в систему и подключается к комнате по коду.
5. Организатор запускает игру, вопросы и таймер синхронно показываются всем участникам.
6. После завершения игры отображается лидерборд, а результаты сохраняются в историю.

## Стек
- `client`: React + Vite + TypeScript + Socket.IO client
- `server`: Node.js + Express + Prisma + PostgreSQL + Socket.IO
- `infra`: Docker Compose

## Запуск через Docker
1. Убедитесь, что установлен Docker Desktop.
2. Выполните:

```bash
docker compose up --build
```

3. После старта сервисы будут доступны по адресам:
- клиент: `http://localhost:4173`
- API: `http://localhost:4001`
- PostgreSQL: `localhost:5433`

Если нужный порт уже занят, можно переопределить его перед запуском:

```powershell
$env:API_PORT=4010
$env:CLIENT_PORT=4174
$env:POSTGRES_PORT=5440
docker compose up --build
```

При первом запуске сервер автоматически:
- применяет Prisma migrations
- при необходимости синхронизирует схему базы
- проверяет, пуста ли база
- добавляет демо-данные, если база новая

## Демо-аккаунты
- организатор: `organizer@quizzy.local` / `demo12345`
- игрок: `player@quizzy.local` / `demo12345`

Эти учётные записи и значения секретов ниже предназначены только для локального демо-режима и не должны использоваться в production. Перед публичным деплоем замените секреты и отключите демонстрационные доступы.

## Локальная разработка без Docker
1. Установите зависимости:

```bash
npm install
```

2. Поднимите PostgreSQL и создайте `.env` на основе `server/.env.example`.
   Клиент для локальной разработки по умолчанию ожидает API на `http://localhost:4000`. Если у вас другой адрес, создайте `client/.env` на основе `client/.env.example`.
3. Сгенерируйте Prisma Client и примените миграции:

```bash
npm run prisma:generate
npm run prisma:migrate
npm run prisma:seed
```

4. Запустите клиент и сервер:

```bash
npm run dev
```

## Замечания
- Изображения вопросов сохраняются локально в volume Docker.
- Маршрутизация клиента использует `HashRouter`, поэтому статический preview-сервер работает без дополнительной настройки nginx.
- Для MVP сервер является единственным источником истины: баллы, комнаты и текущий вопрос больше не живут только в браузере.
